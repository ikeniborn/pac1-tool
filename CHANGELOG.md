# Changelog

Each hardcoded fix gets a sequential label `FIX-N` in code comments.

- FIX-377 (grounding_refs hard-gate в evaluator): `agent/evaluator.py` + `tests/regression/test_grounding_refs_validation.py`. Диагноз последнего прогона: t23/t40 фейлились с harness-ошибкой `answer missing required reference 'accounts/acct_009.json'` / `contacts/mgr_002.json` — агент упоминал ID в `report.message`, но не приклеивал соответствующий путь в `report.grounding_refs`. Skeptic-evaluator пропускал такие ответы (LLM не верифицировал ID-coverage). Новая helper `validate_grounding_refs(report)` — generic regex `\b[a-z]{2,5}_\d{2,4}\b` (case-insensitive) извлекает все vault-ID из message; для каждого требует хотя бы один путь в `grounding_refs`, содержащий ID как substring. Все ID покрыты → pass; partial coverage → reject со списком missing IDs в `correction_hint` («Add the source file path(s) to grounding_refs.»). Сообщение без ID-pattern → skip (no false positives на нейтральных ответах). Интеграция: pre-check в `evaluate_completion()` ДО любого DSPy/LLM-вызова — экономит токены и даёт детерминистичный verdict. Reuse существующего retry-механизма `st.eval_rejections < _MAX_EVAL_REJECTIONS` в `loop.py` — агенту прилетает суггестия дополнить grounding_refs. Tests: 6 TC покрытия (missing/single-covered/multi-partial/multi-full/no-IDs/case-insensitive) + integration test, проверяющий что `call_llm_raw` не вызывается при rejection.

- FIX-377 (purge research contamination tooling): `scripts/purge_research_contamination.py` + `tests/regression/test_purge_script.py`. Reflector ошибочно записал в `data/wiki/graph.json` ~467/1657 узлов из артефактов harness-ошибок (INVALID_ARGUMENT, OTP, seq.json increment, "skipping the actual email", reporttaskcompletion call rejected/returned ERROR, completion call rejected, answer was already, must populate required fields, artifact id) — это шум, который раздувает retrieval-bandwidth и приводит reflector в локальный минимум. Скрипт сканирует graph.json, находит узлы где `text + tags` содержит хотя бы один keyword из настраиваемого списка (default — 10 строк выше), dry-run по умолчанию. С `--apply`: (1) перемещает узлы в `data/wiki/graph_archive.json` с `confidence=0`, (2) удаляет рёбра, ссылающиеся на удалённые узлы, (3) сканирует `data/wiki/pages/*.md` и удаляет блоки `## Successful pattern:` / `## Verified refusal:` если содержимое блока содержит keyword (граница блока — следующий `## ` или EOF), (4) полностью удаляет `data/wiki/fragments/research/`. Idempotent — повторный `--apply` no-op. Скрипт не вызывается автоматически (ни в hooks, ни в Makefile, ни в CI) — только ручной запуск после ревью dry-run. Tests: 4 регрессии в `tests/regression/test_purge_script.py` через `tmp_path` fixture (TC1 dry-run находит 3/5 кандидатов, TC2 apply переносит 3 в archive и подрезает edges, TC3 idempotency, TC4 page-block removal). Default keywords: `invalid_argument`, `otp`, `artifact id`, `seq.json increment`, `must populate required fields`, `skipping the actual email`, `reporttaskcompletion call rejected`, `reporttaskcompletion returned error`, `completion call rejected`, `answer was already`. CLI: `--keywords K1 K2 ...`, `--graph-path`, `--archive-path`, `--pages-dir`, `--fragments-dir`, `--apply`.
- FIX-376 (researcher remediation, eight env-gated knobs + global escape ladder, all default OFF): целенаправленный фикс non-convergence на t11/t19/t43-классе задач. Все изменения researcher-only — normal mode не затронут. **(a) Evaluator fail-closed** (`agent/evaluator.py`, `agent/researcher.py::_evaluator_gate`): новый kwarg `fail_closed: bool=False` в `evaluate_completion()`. При `RESEARCHER_EVAL_FAIL_CLOSED=1` ошибка LLM/parse внутри гейта возвращает `EvalVerdict(approved=False, correction_hint="EVAL_ERROR")` и researcher интерпретирует это как inconclusive — НЕ инжектит `EVAL_REJECTED` (избегает ложного следа), НЕ short-circuit'ит, увеличивает `stats["researcher_eval_inconclusive"]` и continue'ит. До фикса fail-open silently approve-ил при transient LLM-сбоях. **(b) Hint forcing** (`agent/researcher.py` + `agent/loop.py::run_loop`): когда подряд cycles инжектят один и тот же тип хинта (`OUTCOME_FLIP_HINT`/`EVAL_REJECTED`/`DRIFT`/`REFUSAL_RETRY`/`MIDCYCLE_ABORTED`) и агент его игнорирует, эскалация из system-addendum в mandatory user-message. Cap `RESEARCHER_HINT_MAX_INJECTIONS=2` подряд. Stats: `researcher_hint_forcing_injected`. Env: `RESEARCHER_HINT_FORCING=0`. **(c) Mid-cycle breakout** (`agent/researcher.py::_midcycle_breakout`, `agent/loop.py`): callback-based checkpoint каждые `RESEARCHER_MIDCYCLE_CHECK_EVERY=5` шагов. Триггеры: same `(tool, path)` ≥ `RESEARCHER_MIDCYCLE_REPEAT_THRESHOLD=3` ИЛИ нет новых `step_fact.kind` в trailing slice. На abort — выход inner-loop, `MIDCYCLE_ABORTED` маркер в `hypothesis_for_next` следующего цикла. Stats: `researcher_midcycle_aborts`. Env: `RESEARCHER_MIDCYCLE_BREAKOUT=0`. **(d) Reflector diversify** (`agent/reflector.py::reflect`, `agent/researcher.py::_is_monotonic_hypothesis`): `reflect()` теперь принимает `prior_hypotheses: list[str] | None`, в user_msg блок «You have already proposed: [...]. Produce a MATERIALLY DIFFERENT hypothesis or output `stuck_converged`». Monotonicity detector ужесточён: при `RESEARCHER_REFLECTOR_DIVERSIFY=1` mean pairwise Jaccard threshold 0.45 (вместо 0.6), K=2 identical (Jaccard=1.0) безусловно monotonic. Env: `RESEARCHER_REFLECTOR_DIVERSIFY=0`, `RESEARCHER_REFLECTOR_PRIOR_WINDOW=3`. **(e) Adaptive step budget + global cap (escape ladder)** (`agent/researcher.py::_compute_step_budget`, `agent/loop.py` soft-stall): cycles 1-2 → `RESEARCHER_STEPS_PER_CYCLE`. Cycle ≥3: если prior outcome=stuck AND prior_steps_used ≥ 80% prior_budget → `min(RESEARCHER_STEPS_MAX=30, prior_steps_used × 1.5)`. **Global cap**: `total_steps_used >= RESEARCHER_TOTAL_STEP_BUDGET=180` → `stats["researcher_global_step_cap_hit"]=True`, accept best-known reflection.final_answer (или последний `pending_refusal`), break outer loop. Без cap'а runaway-цикл с adaptive=1 теоретически 300 шагов (10 × 30) — global cap делает escape ladder finite. Soft-stall (`RESEARCHER_SOFT_STALL=1`): stall-detector в advisory-only режиме — записывает `step_fact.kind="stall_advisory"`, не triggers `_handle_stall_retry`. Stats: `researcher_step_budget_per_cycle`, `researcher_total_steps_used`, `researcher_global_step_cap_hit`, `researcher_soft_stall_hints`. Env: `RESEARCHER_STEPS_ADAPTIVE=0`, `RESEARCHER_STEPS_MAX=30`, `RESEARCHER_TOTAL_STEP_BUDGET=180`, `RESEARCHER_SOFT_STALL=0`. **(f) Dynamic refusal budget** (`agent/researcher.py`): фиксированный `RESEARCHER_REFUSAL_MAX_RETRIES=3` заменён на `cap = max(RESEARCHER_REFUSAL_MIN_CYCLES_LEFT=2, max_cycles - cycle - 1)` при `RESEARCHER_REFUSAL_DYNAMIC=1` — refusals не пре-эмптят 10-цикловый бюджет, агент получает достаточно retries для исследования alt-интерпретаций. Last-chance hint теперь подтягивает top-K bullets из `_load_negative_warnings(task_type, top_k=2)` (FIX-370 reuse) и приклеивает как «alternative interpretations seen elsewhere: ...» к flip-hint. Env: `RESEARCHER_REFUSAL_DYNAMIC=0`, `RESEARCHER_REFUSAL_MIN_CYCLES_LEFT=2`. **(g) Graph quarantine** (`agent/wiki_graph.py::retrieve_relevant`, `agent/researcher.py`): три новых kwargs — `min_retrieve_confidence`, `degraded_this_session`, `quarantine_weak_antipatterns`. Скоринг скипает узлы с conf < `RESEARCHER_GRAPH_MIN_CONF=0.35`, узлы degraded в текущем trial'е, weak antipatterns (`uses<2 AND conf<0.5`). Researcher отслеживает `degraded_this_session: set[str]`. Defaults permissive — нет регрессии в normal mode. Stats: `researcher_graph_quarantined_nodes`. Env: `RESEARCHER_GRAPH_QUARANTINE=0`, `RESEARCHER_GRAPH_MIN_CONF=0.35`. **(h) Full-trace drift** (`agent/researcher.py::_lcs_ratio`, `_detect_drift_lcs`): после prefix-check (FIX-372) — fallback на LCS-ratio между full current trajectory и каждым successful pattern. Все ratios < `RESEARCHER_DRIFT_LCS_MIN=0.4` → drift hint с аннотацией divergence-индекса (первый шаг где совпадение прерывается). Prefix-check сохраняется как fast-path. Stats: `researcher_drift_lcs_hints`. Env: `RESEARCHER_DRIFT_FULL_TRACE=0`, `RESEARCHER_DRIFT_LCS_MIN=0.4`. **Verification:** все env=0 (default) → нет изменений в normal mode и researcher baseline; полный full-ON конфиг включается через `RESEARCHER_EVAL_FAIL_CLOSED=1 RESEARCHER_HINT_FORCING=1 RESEARCHER_MIDCYCLE_BREAKOUT=1 RESEARCHER_REFLECTOR_DIVERSIFY=1 RESEARCHER_STEPS_ADAPTIVE=1 RESEARCHER_SOFT_STALL=1 RESEARCHER_REFUSAL_DYNAMIC=1 RESEARCHER_GRAPH_QUARANTINE=1 RESEARCHER_DRIFT_FULL_TRACE=1`.
- FIX-375b (researcher OK-loop hardening, follow-up к FIX-375): smoke-run на t11/t19/t43 показал что OK-flip (3a) на t43 не срабатывает — evaluator gate был gated на `reflection.is_solved=True`, а reflector 15 циклов подряд выдавал outcome="stuck" даже когда агент сам репортил `OUTCOME_OK` с одним и тем же ответом. Три связанных изменения. **(A) evaluator-primary gate** (`agent/researcher.py`): `_gate_relevant` снимает `reflection.is_solved` из условия — evaluator теперь судит любой `OUTCOME_OK`. Без этой памяти reflector судит каждую траекторию изолированно — не замечает что агент уже 14 раз дал тот же ответ. researcher.py трекает `_outcome_history` и передаёт его в reflect. **(C) hard guard на consecutive OK loop**: новый env `RESEARCHER_OK_LOOP_LIMIT` (default 5). - FIX-375 (researcher OUTCOME_FLIP_HINT + diversification detector): `agent/researcher.py`. Диагноз по прогону 10 задач: t11/t19 корректно сходились на refusal, но benchmark ожидал OK (reflector перепробовал разнообразные гипотезы — write/schema/artifact — и честно упёрся); t43 упорно выдавал OUTCOME_OK с неверной интерпретацией 15 циклов подряд, evaluator rejected каждый раз с одинаковым reason («no exact match»), но reflector.hypothesis_for_next оставался монотонным (все DRIFT-ссылки на одну t34, trajectory всегда `list→report`). Два симметричных механизма. (a) **OK-side flip** — в ветке evaluator reject (`RESEARCHER_EVAL_GATED=1`) после ≥2 rejections с reason-similarity ≥ `RESEARCHER_FLIP_REASON_SIMILARITY_THRESHOLD` (default 0.5, Jaccard на нормализованных токенах) ИЛИ при монотонности последних `K+1` hypothesis_for_next (default K=2, sim ≥ 0.6) — добавляется `OUTCOME_FLIP_HINT: You've proposed OUTCOME_OK multiple times... consider OUTCOME_NONE_CLARIFICATION/UNSUPPORTED as a valid answer`. Инжектируется как suffix к EVAL_REJECTED-хинту в `reflection.hypothesis_for_next` → попадает в аддендум следующего цикла через reflector. (b) **Refusal last-chance** — после `RESEARCHER_REFUSAL_MAX_RETRIES` exhausted вместо сразу accept выдаём один extra цикл с зеркальным хинтом (`RESEARCHER_REFUSAL_LAST_CHANCE=1`, default on): `OUTCOME_FLIP_HINT: You've refused N times... if there's ANY plausible interpretation where task IS answerable, attempt now`. Маркер использования — `stats["researcher_refusal_last_chance_used"]`, чтобы не зацикливаться. Helpers: `_jaccard(a,b)`, `_is_monotonic_hypothesis(history, k, threshold)`. Tracking — локальные `_hypothesis_history` и `_eval_reject_reasons` per `run_researcher` invocation; гипотеза захватывается СРАЗУ после reflect(), до любых mutation'ов (drift/retry/eval), чтобы детектор мерил reflector, а не наши же суффиксы. Stats: `researcher_flip_hints_injected` (cumulative counter). Env: `RESEARCHER_FLIP_HINT_ENABLED=1`, `RESEARCHER_FLIP_REASON_SIMILARITY_THRESHOLD=0.5`, `RESEARCHER_FLIP_HYP_MONOTONIC_K=2`, `RESEARCHER_FLIP_HYP_SIMILARITY_THRESHOLD=0.6`, `RESEARCHER_REFUSAL_LAST_CHANCE=1`. Self-consistency voting рассмотрен и отвергнут — 3× reflector-токенов на каждом цикле, не решает monotonicity в детерминистичном режиме.
