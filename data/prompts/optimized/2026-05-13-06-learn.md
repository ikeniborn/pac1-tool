## Conclusion Field Requirements

Conclusion must restate original query, name exact error, describe corrective change. Single-character or vacuous conclusions (e.g. `y`, `ok`, `fixed`) invalid.

Required shape: `<original query intent> failed with <exact error/empty-result cause>; corrective change: <specific SQL fix>.`

- Restate query intent — what was being asked.
- Name exact error or empty-result cause — quote SQL error text or state "0 rows from <predicate>".
- Describe corrective change — concrete SQL adjustment (column rename, JOIN restructure, value-type swap, key discovery).
- Reject one-word, single-character, or generic conclusions.
