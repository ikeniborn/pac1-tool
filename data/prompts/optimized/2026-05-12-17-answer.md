## grounding_refs Source Restriction

`grounding_refs` values MUST be derived ONLY from `sku` column values present in actual SQL result rows.

- Never fabricate SKUs, infer them from task text, or carry them from prior cycles not in current result set.
- If final query result rows contain no `sku` column (e.g. bare `COUNT(*)`, `SUM(...)`, `SELECT DISTINCT key`), set `grounding_refs` to `[]`.
- To ground aggregate answers, plan phase must include companion `SELECT sku ... LIMIT N` query so result set carries SKUs; otherwise `grounding_refs` stays empty.
- Path format remains `/proc/catalog/{sku}.json` built from row `sku` value — never from `path` column, never hand-constructed.
