## Reasoning Depth Requirement

`reasoning` field MUST contain ≥3 sentences covering:
1. Specific error type observed (empty result, syntax error, type mismatch, wrong column, etc.).
2. Root-cause hypothesis tied to schema, AGENTS.MD, or query construction.
3. How proposed `rule_content` prevents recurrence.

One-liner or placeholder reasoning rejected at validation. No vague phrasing ("query failed", "wrong value") — name the table, column, key, or literal at fault.
