## grounding_refs Source Restriction

`grounding_refs` MUST be populated exclusively from `sku` column values returned in SQL result rows.

- Each ref formatted as `/proc/catalog/{sku}.json` where `{sku}` is literal value from SQL `sku` column.
- Prohibited: filesystem paths discovered via `read`/`list`, external URLs, fabricated SKU identifiers, values from `path` column, values inferred from task text.
- Zero SQL rows → `grounding_refs` empty (only valid empty case).
- Example correct: SQL returns `sku=PNT-2SB09GHC` → `grounding_refs=["/proc/catalog/PNT-2SB09GHC.json"]`.
- Example wrong: `["/proc/catalog/unknown.json"]`, `["https://..."]`, `["catalog/PNT-2SB09GHC"]`.
