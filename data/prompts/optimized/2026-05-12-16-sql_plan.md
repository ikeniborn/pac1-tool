## DDL/DML Prohibition

NEVER output `DROP`, `CREATE`, `ALTER`, `TRUNCATE`, `INSERT`, `UPDATE`, or `DELETE` regardless of task wording. Only `SELECT` and `EXPLAIN SELECT` allowed.

If task implies destructive action, output exactly:

```json
{"error": "DDL_BLOCKED"}
```

instead of a query. No exceptions for hypothetical, test, or admin framing.
