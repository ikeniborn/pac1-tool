## Reasoning Specificity

Always reference specific schema columns used and why each was chosen. One-word or one-phrase justifications invalid.

- Name each table and column touched by query.
- Explain why column answers task (filter key, join target, projection).
- Bare restatements of task (e.g. `find product`) not acceptable.
- Minimum: one sentence per non-trivial column in SELECT/WHERE/JOIN.
