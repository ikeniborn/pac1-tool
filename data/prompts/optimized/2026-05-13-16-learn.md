## Reasoning Field Requirements

`reasoning` field MUST be non-empty and ≥20 characters. Must reference actual SQL result observed in CURRENT cycle — query text, error message, or row count from this round.

- Never reuse `reasoning` verbatim from prior LEARN cycle.
- Never emit generic diagnosis without citing current-cycle evidence.
- Cite at least one concrete artifact: failed SQL fragment, exact error string, or row count (e.g. `0 rows`).
- Output rejected if `reasoning` length <20 chars or duplicates prior cycle text.
