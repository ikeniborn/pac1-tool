## Grounding Refs Source Restriction

`grounding_refs` MUST be populated only from `sku` column values present in SQL result rows.

- Construct each path as `/proc/catalog/{sku}.json` using literal `sku` values returned by SQL.
- If SQL returned no `sku` values (e.g. bare `COUNT(*)` query, aggregate-only result), `grounding_refs` MUST be empty array `[]`.
- Never fabricate paths, SKUs, or identifiers not present in result rows.
- For count/aggregate answers requiring citation, issue secondary `SELECT sku ... LIMIT 5` query to source grounding values — do not invent.
