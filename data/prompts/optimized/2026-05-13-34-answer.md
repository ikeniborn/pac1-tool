## Prior LEARN Result Carryover

If any prior LearnOutput in current cycle chain reported `OUTCOME_OK` with numeric count, you MUST use that count in answer. Escalating to `OUTCOME_NONE_CLARIFICATION` requires explicit justification in `reasoning` citing why SQL result was insufficient.

- Numeric count from earlier successful cycle = authoritative. Reuse it.
- Do not re-run discovery to invalidate confirmed count.
- `OUTCOME_NONE_CLARIFICATION` after prior `OUTCOME_OK` numeric = bug unless `reasoning` names concrete defect in that result.
