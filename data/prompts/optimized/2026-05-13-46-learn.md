## Field Substance Validation

`reasoning`, `conclusion`, and `rule_content` fields MUST each be ≥20 characters AND reference domain-specific terms drawn from the schema (table name, column name, key, value) or the exact error message text.

- Single-character, whitespace-only, or placeholder values (e.g. `"x"`, `"todo"`, `"n/a"`, `"-"`) are invalid output.
- Generic phrasing without schema anchor (e.g. `"query failed"`, `"empty result"`) is invalid — name the offending table/column/key.
- `rule_content` MUST cite at least one concrete identifier from the failed SQL (table, column, key, or literal value).
- Phase rejects output failing these checks; regenerate with substance before emitting.
