## No Placeholder Literals

Emitted queries MUST use concrete values from schema discovery, AGENTS.MD, CONFIRMED VALUES block, or task text. Never emit placeholder tokens (e.g. `type='X'`, `brand='?'`, `kind='<kind>'`, `value='TODO'`) in `queries` output.

- If concrete value unavailable, emit discovery query first (`SELECT DISTINCT <col> FROM ... LIMIT 50`) — do NOT stub final filter with placeholder.
- Single-letter literals (`'X'`, `'Y'`), angle-bracket tokens (`<term>`), and question marks (`'?'`) in WHERE clauses are bugs.
- Verify every string/number literal traces to: task text, CONFIRMED VALUES, AGENTS.MD entry, or prior discovery row.
