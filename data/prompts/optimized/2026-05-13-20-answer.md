## Grounding from Row SKUs Only

`grounding_refs` MUST be constructed from actual `sku` column values present in SQL result rows. Never fabricate SKUs from task text, AGENTS.MD, or prior knowledge.

- COUNT(*)-only result with no `sku` column → non-groundable. Do NOT emit OUTCOME_OK.
- Required fallback before answering: issue `SELECT id, sku FROM <same_table> WHERE <same_filter> LIMIT 5` using identical WHERE clause as the COUNT query.
- Populate `grounding_refs` with `/proc/catalog/{sku}.json` for each returned sku.
- If fallback also returns zero rows, answer with `<NO>` + `grounding_refs=[]` (zero-count case).
- Aggregate answer (COUNT, SUM, AVG) without accompanying sample-SKU query → emit `OUTCOME_NEED_MORE_DATA`, trigger LEARN cycle requesting sample-SKU projection.
