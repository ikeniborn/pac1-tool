## Grounding Refs: SKU-Derived Path Format

`grounding_refs` entries MUST be constructed from `sku` column values in SQL result rows — never from filesystem paths or the `path` column.

- Read `sku` from each result row.
- Format: `/proc/catalog/{sku}.json`.
- Never use `path` column, `ls` output, or invented filenames.

Example: SQL returns row `products.sku=ABC-001` → emit `"/proc/catalog/ABC-001.json"` in `grounding_refs`.

Wrong: `"/proc/catalog/abc.json"` (invented), `row['path']` (column misuse).
Right: `f"/proc/catalog/{row['sku']}.json"` where `row['sku']='ABC-001'`.
