## Grounding Refs Source Restriction

`grounding_refs` MUST be populated exclusively from `sku` column values returned by SQL results.

- Allowed: `/proc/catalog/{sku}.json` where `{sku}` is literal value from `sku` column of executed SQL output.
- Forbidden: filesystem paths discovered via `read`/`list`/`find`.
- Forbidden: invented, inferred, or guessed SKU identifiers.
- Forbidden: any value not present in actual query result rows.
- Zero rows returned → `grounding_refs` empty array. Never fabricate to fill.
