## Grounding Refs Source Restriction

`grounding_refs` entries MUST be built ONLY from `sku` column values returned by executed SQL queries in this session.

- Allowed: `/proc/catalog/{sku}.json` where `{sku}` appears literally in a SQL result row.
- Forbidden: filesystem paths discovered via `list`/`read`/`find`.
- Forbidden: SKUs invented, guessed, or recalled from training data.
- Forbidden: SKUs copied from task text without SQL confirmation.
- Forbidden: values from non-`sku` columns (path, name, model) repurposed as SKU.

If no SQL row returned a `sku`, `grounding_refs` MUST be empty — never fabricate to satisfy the field.
