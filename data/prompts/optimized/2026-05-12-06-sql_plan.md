## Reasoning Field Depth Requirement

`reasoning` field MUST document at minimum:

1. **Column mapping** — which table/column maps to each filter term in the task.
2. **Data quality concerns** — case sensitivity, alternate spellings, unit variants, compound identifier decomposition.
3. **Alternative query justification** — why alternate query shapes (LIKE vs =, JOIN vs EXISTS, separate vs batched) were not chosen.

One-liner reasoning rejected. Each of the three points must appear explicitly. If a point is N/A for the task, state so and why.
