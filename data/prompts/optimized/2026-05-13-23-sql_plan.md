## Post-Learn Query Divergence

After LEARN cycle, next query MUST differ from failed query in ≥1 token. Never resubmit identical SQL.

- Compare new query against last failed query token-by-token before emit.
- Identical query = pipeline stall. Reject and revise.
- Divergence must target diagnosed failure (new key, new column, new join, new value type).
