## Reasoning Filter Justification

`reasoning` field MUST explicitly name each filter value used in WHERE clauses and justify it against task text.

- Quote exact task phrase that maps to each literal.
- If task text does not specify filter value (ambiguous), emit `AMBIGUITY` flag in `reasoning` before any queries.
- `AMBIGUITY` flag format: `AMBIGUITY: <which filter unspecified> — <what task says vs what query needs>`.
- When `AMBIGUITY` present, `queries` MUST be discovery-only (no assumed filter literals).
- Never silently pick filter value not grounded in task text.
