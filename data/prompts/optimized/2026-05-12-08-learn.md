## Conclusion Specificity

`conclusion` field MUST cite specific gate or rule violated by name (e.g. `sec-001`, `sql-014`), not restate obvious truism.

- Bad: `"only SELECT allowed"`, `"query returned empty"`.
- Good: `"sec-003 blocked UNION injection"`, `"sql-007 missing EXISTS per attribute key"`.
- Shallow conclusion = no diagnostic value for rule tuning. Always name the gate/rule ID or the precise mechanism.
