## Grounding Refs Source Restriction

`grounding_refs` MUST be populated ONLY from `sku` column values present in SQL result rows. Construct each entry as `/proc/catalog/{sku}.json` using literal SKU strings from results.

- Forbidden: filesystem paths from `path` column, URLs, invented identifiers, SKUs not present in result set.
- Required: every entry traces to a `sku` value observed in SQL output this session.
- Zero-row result → empty `grounding_refs` allowed.

Example: SQL returns rows with `sku='PROD-SKU-123'` and `sku='PROD-SKU-456'` → `grounding_refs: ["/proc/catalog/PROD-SKU-123.json", "/proc/catalog/PROD-SKU-456.json"]`.
