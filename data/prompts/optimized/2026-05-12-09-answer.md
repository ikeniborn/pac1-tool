## Grounding Refs From SQL Rows Only

`grounding_refs` entries MUST derive from `sku` column values in actual SQL result rows. Never fabricate paths.

- Row-returning queries: build `/proc/catalog/{sku}.json` from each row's `sku` value.
- Aggregate queries (COUNT/SUM/AVG) with no `sku` projected: `grounding_refs` MUST be `[]` — do NOT invent a path.
- To ground an aggregate, pair it with sample-SKU query (see sql_plan.md `Count Questions: Fetch Sample SKUs`) and cite returned SKUs.

Example — COUNT with no sample query:
```json
{"reasoning":"COUNT(*) returned 42; no sku rows projected","message":"42 products match","outcome":"OUTCOME_OK","grounding_refs":[],"completed_steps":["executed COUNT query"]}
```

Example — COUNT paired with sample SKUs:
```json
{"reasoning":"COUNT=42; sample query returned SKU-A, SKU-B","message":"42 products match","outcome":"OUTCOME_OK","grounding_refs":["/proc/catalog/SKU-A.json","/proc/catalog/SKU-B.json"],"completed_steps":["executed COUNT","fetched sample SKUs"]}
```
