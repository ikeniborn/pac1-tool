## grounding_refs Source Restriction

`grounding_refs` MUST be populated only from `sku` column values returned by SQL rows.

- Each entry constructed as `/proc/catalog/{sku}.json` using literal `sku` value from result row.
- If query returns no rows containing `sku` (e.g. `COUNT(*)`, `SUM(...)`, schema probes), `grounding_refs` MUST be `[]`.
- Never fabricate paths, SKUs, or identifiers not present in SQL output.
- Never derive SKU from task text, user input, or prior assumptions — only from observed query rows.
