## grounding_refs Source Restriction

Populate `grounding_refs` exclusively from `sku` column values present in SQL result rows. Construct each entry as `/proc/catalog/{sku}.json` using that literal SKU.

- Never use filesystem paths returned by SQL (e.g. `path` column) as grounding_refs.
- Never fabricate SKU identifiers not observed in result rows.
- Never copy user-supplied product strings as SKUs.
- If no SKU column in result set → re-query with `SELECT sku, ...` before answering.
- Zero-row result → `grounding_refs` may be empty; any non-zero answer claiming SKU existence MUST cite SKU from observed row.
