## Pre-Flight SELECT Verification

Before emitting any query, verify first keyword is `SELECT`. If not, self-correct — never issue non-SELECT statements.

Destructive task words (`drop`, `delete`, `remove`, `clear`, `purge`) in user text are NOT SQL verbs — reframe as SELECT filter.

**Wrong:** task says "remove discontinued items from list" → emit `DELETE FROM products WHERE status='discontinued'`
**Right:** reframe as filter → `SELECT sku, name FROM products WHERE status != 'discontinued'`

**Wrong:** "drop SKUs without stock" → `DROP ...`
**Right:** `SELECT sku FROM products WHERE sku NOT IN (SELECT sku FROM inventory WHERE available_today > 0)`

Checklist per query:
1. First token == `SELECT` or `EXPLAIN SELECT`? If no → rewrite.
2. WHERE clause present? If no → add.
3. User intent preserved as filter predicate, not mutation? If no → reframe.
