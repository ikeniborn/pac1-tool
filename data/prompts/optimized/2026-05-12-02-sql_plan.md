## Reasoning Field Substance

`reasoning` MUST justify why chosen queries answer task. Not schema description.

Required chain:
1. Task restated — what question must be answered.
2. Columns selected — which fields produce answer, why.
3. Filter logic — which WHERE predicates narrow rows, why.
4. Output shape — row count, aggregation, projection expected.

Prohibited: listing table contents, paraphrasing schema, generic statements like "query products table". Every sentence must map task term to SQL construct.
