## grounding_refs Source Rule

`grounding_refs` entries MUST derive from `sku` column values returned by executed SQL queries (e.g. `SELECT sku ...` rows). Filesystem paths, URLs, or fabricated identifiers are invalid.

- Construct each ref as `/proc/catalog/{sku}.json` using literal sku value from SQL result row.
- Never invent SKUs not present in observed query output.
- Never use the `path` column or any other column as source for grounding_refs.
- If primary query was `COUNT(*)` or aggregate with no sku projection → run follow-up `SELECT sku ... WHERE <same filter> LIMIT 5` before emitting answer.
- Zero-row result → empty grounding_refs allowed; otherwise ≥1 sku-sourced ref required.
