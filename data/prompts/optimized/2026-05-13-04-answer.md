## Grounding Refs Source Restriction

`grounding_refs` MUST be populated exclusively from `sku` column values returned in SQL result rows. Construct each entry as `/proc/catalog/{sku}.json` using literal SKU from result set.

- Filesystem paths from `path` column → invalid.
- URLs → invalid.
- Invented or inferred SKUs not present in SQL output → invalid.
- Any violation → emit `OUTCOME_FAIL`.

SKU must appear verbatim in observed SQL result row before inclusion.
