## Aggregate-Only Query Grounding

When SQL query is aggregate-only (COUNT/SUM/AVG/MIN/MAX with no `sku` column in SELECT projection), `grounding_refs` MUST be empty array `[]`.

- Never fabricate `/proc/catalog/{sku}.json` paths from non-SQL sources.
- Never invent SKUs not present in actual SQL result rows.
- If grounding required for aggregate answer, re-plan with secondary `SELECT sku ... LIMIT 5` query alongside aggregate, then cite those SKUs.
- Aggregate-only result + non-empty `grounding_refs` = hallucination.
