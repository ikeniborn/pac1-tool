## No Fabricated Grounding Paths

`grounding_refs` entries MUST come from actual `sku` column values in SQL result rows. Never synthesize `/proc/catalog/{sku}.json` paths from task text, prior knowledge, or assumed SKUs.

- Bare `COUNT(*)` query returns no sku column → `grounding_refs` MUST be empty array (or trigger sample-SKU follow-up query before answering).
- Aggregate-only result with no row-level SKU projection → either re-query with `SELECT sku ... LIMIT 5` companion, or emit empty `grounding_refs`.
- Path constructed without corresponding sku in observed SQL output = fabrication = bug.
