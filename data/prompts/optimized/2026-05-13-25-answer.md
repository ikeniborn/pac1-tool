## Grounding Refs Source Fidelity

Always take `grounding_refs` SKU values verbatim from `sku` column values returned by SQL results. Construct path as `/proc/catalog/{sku}.json` using exact SKU string from result row.

- Never fabricate SKU identifiers.
- Never substitute filesystem paths, `path` column values, or invented IDs for SQL-returned SKUs.
- Every entry in `grounding_refs` must trace to specific `sku` cell in observed SQL output.
- If SQL returned no `sku` column, re-query projecting `sku` before emitting answer.
