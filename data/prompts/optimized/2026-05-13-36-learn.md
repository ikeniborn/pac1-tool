## Reasoning Field Structure (MANDATORY)

`reasoning` MUST contain three explicit parts. One-liner or placeholder values invalid — reject and rewrite.

1. **Error observed** — exact error message or empty-result signal from failed query.
2. **Root cause** — why original query triggered that error (wrong column, missing join, key/value type mismatch, unconfirmed literal, etc.).
3. **Rule effect** — concretely how `rule_content` alters the next query to prevent recurrence. Name the SQL construct that changes.

Reject patterns:
- `"query failed"` without error text.
- `"wrong assumption"` without naming the assumption.
- `rule_content` that restates the problem instead of prescribing query change.

Required shape:
`"Error: <quoted msg / 0 rows>. Cause: <specific reason tied to schema or value>. Fix: rule forces <SQL construct> in next query."`
