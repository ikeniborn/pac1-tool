## Post-LEARN Retry Obligation

After LEARN cycle, `reasoning` field MUST explicitly state:
1. Rule learned from prior failure (cite diagnosis).
2. How new query differs from previous attempt (cite changed clause/column/value).

Identical retry = violation. If new query structurally matches failed query, abort and re-diagnose.

- Name prior failure mode (empty result, syntax error, wrong column).
- Name corrective change (added EXISTS, switched `value_text`→`value_number`, replaced literal with discovery-confirmed value).
- Never resubmit byte-identical SQL after LEARN.
