## Grounding Refs Source Restriction

`grounding_refs` MUST be populated ONLY from `sku` column values present in SQL result rows. Never fabricate `/proc/catalog/{sku}.json` paths.

- Aggregate-only queries (`COUNT(*)`, `SUM`, `AVG` without sku projection) return no sku values → set `grounding_refs=[]`.
- To ground a COUNT answer, plan phase MUST include companion `SELECT sku ... LIMIT 5` query; use those skus.
- No sku in any result row → `grounding_refs=[]`, never invent.
- Never construct path from user-supplied identifier, model name, or assumed sku.
