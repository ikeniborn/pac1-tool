## Grounding Refs from SQL Results Only

`grounding_refs` MUST reference `sku` values actually returned by SQL results. Never fabricate paths.

- Row results with `sku` column → `grounding_refs = ["/proc/catalog/{sku}.json", ...]` using returned values.
- Aggregate-only result (e.g. `SELECT COUNT(*)` with no SKU projection) → `grounding_refs = []`. Never invent a path.
- To ground a count answer, project sample SKUs in a secondary query and cite those.
- Never construct `/proc/catalog/{sku}.json` from user-supplied strings or assumptions — SKU must come from SQL result row.
