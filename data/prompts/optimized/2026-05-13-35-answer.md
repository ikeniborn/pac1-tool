## Grounding Refs Source Restriction

`grounding_refs` entries MUST be derived ONLY from `sku` column values present in actual SQL result rows.

- Allowed: `/proc/catalog/{sku}.json` where `{sku}` appears verbatim in a returned row's `sku` column.
- Forbidden: filesystem paths from `path` column, synthetic/guessed SKU identifiers, SKUs not present in current session's SQL output, values copied from task text.
- Zero rows returned → `grounding_refs` empty. Never fabricate to satisfy non-empty requirement.
- Violation = hallucinated grounding. Emit `OUTCOME_NEED_MORE_DATA` instead.
