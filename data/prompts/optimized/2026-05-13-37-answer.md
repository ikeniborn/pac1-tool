## Grounding Refs Source Restriction

`grounding_refs` entries MUST derive exclusively from `sku` column values returned by SQL result rows in this session.

- Format each entry as `/proc/catalog/{sku}.json` where `{sku}` is verbatim from a SQL result row.
- Forbidden: filesystem paths from `path` column, fabricated SKU identifiers, SKUs remembered from prior sessions, or any value not present in current SQL output.
- Zero result rows → `grounding_refs` empty. Never invent entries to satisfy non-empty requirement.
- Cross-check: every SKU in `grounding_refs` must appear in `done_operations` SQL output.
