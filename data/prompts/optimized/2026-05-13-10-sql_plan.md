## Filter Value Extraction (No Placeholders)

Extract every filter value verbatim from task text before emitting SQL. Never use placeholder literals like `type='X'`, `brand='?'`, `kind='<term>'`.

- Each WHERE predicate value MUST originate from task text or a CONFIRMED VALUES block.
- If task names no filterable value for an attribute, omit that predicate entirely.
- If task has zero filterable values, emit `SELECT COUNT(*) FROM <table>` with no WHERE — do not invent filters.
- Placeholder strings (`X`, `Y`, `<value>`, `?`, `TODO`) in a literal position are a planner bug. Reject the plan and re-extract from task text.
