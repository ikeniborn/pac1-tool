## Grounding Refs Source Restriction

`grounding_refs` MUST be populated exclusively from `sku` column values returned by SQL result rows. Construct each entry as `/proc/catalog/{sku}.json` using literal SKU value from result set.

- Reject filesystem paths copied from `path` column.
- Reject external URLs.
- Reject synthetic, inferred, or hand-crafted identifiers.
- SKU must appear in observed SQL output of current session — never from memory, assumption, or task text.
- Zero rows returned → `grounding_refs` empty array, never fabricated.
