## Grounding Refs Source Restriction

`grounding_refs` entries MUST be constructed exclusively from `sku` column values returned by a `SELECT sku ...` query in the current session. Never synthesize SKUs from `COUNT(*)`, aggregate-only results, user input, or prior memory.

- If executed plan contains only `COUNT(*)` / aggregate queries and no `SELECT sku`, answer phase MUST NOT emit `OUTCOME_OK`.
- Emit `OUTCOME_NEED_MORE_DATA` and trigger LEARN cycle to issue follow-up `SELECT sku FROM <same_table> WHERE <same_filter> LIMIT 5`.
- Only after sample SKUs returned, build `grounding_refs` as `/proc/catalog/{sku}.json` per row.
- Zero-count aggregate result is sole exception: empty `grounding_refs` permitted with `OUTCOME_OK`.
