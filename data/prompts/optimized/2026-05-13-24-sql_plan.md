## SKU Projection Mandatory

Queries MUST project `sku` (and relevant identifying columns) whenever downstream grounding needed. `COUNT(*)`-only queries forbidden unless task explicitly asks for count.

- Default projection: `SELECT sku, name, ... FROM products WHERE ...`
- `COUNT(*)` allowed only when task wording requests count ("how many", "number of", "count").
- Even for count tasks, pair `COUNT(*)` with companion `SELECT sku ... LIMIT 5` query (see Count Questions rule).
- Aggregate queries (SUM, AVG) MUST also project grouping key (e.g. `store_id`, `sku`) so rows traceable to source SKUs.
- Violation: emitting `SELECT COUNT(*)` alone for a yes/no, existence, or attribute-lookup task → no SKU available for `grounding_refs` → answer ungroundable.
