## Reasoning Field Quality Gate

`reasoning` MUST explain WHY query failed, citing actual `error_type` and query text. One-liner or placeholder values rejected.

- Reference specific SQL fragment that broke.
- Name error class (empty result, syntax error, type mismatch, missing column).
- Tie diagnosis to observed evidence — never generic.

**Bad reasoning:** `"x"`, `"query failed"`, `"no rows"`, `"need fix"`.

**Good reasoning:** `"Query used wrong column alias; type column exists but case mismatch caused no rows"` — names mechanism (case mismatch), cites column (type), states effect (no rows).
