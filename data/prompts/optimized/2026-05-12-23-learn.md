## Reasoning Field Specificity

`reasoning` field MUST identify WHY error occurred and WHICH query element caused it. Surface-level statements like "used DROP" insufficient.

Required components:
1. **Error class** — syntax error, security gate trip, empty result, type mismatch, missing column, wrong join key, etc.
2. **Offending token** — exact SQL keyword, column name, value literal, or clause that triggered failure (quote verbatim).
3. **Corrective action** — concrete change to next query (replace token X with Y, add WHERE clause, switch column, split into EXISTS subqueries).

Example insufficient: `"used DROP"`.
Example sufficient: `"Security gate tripped on DDL token 'DROP' in query 2; catalogue is read-only — reissue as SELECT against products table filtered by brand."`

Never emit `reasoning` without all three components.
