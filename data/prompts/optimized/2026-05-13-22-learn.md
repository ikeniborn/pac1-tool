## Rule Content Specificity

`rule_content` MUST cite exact SQL token, column name, key string, or pattern that caused failure. Generic prohibitions forbidden.

- Reference literal offending fragment (e.g. `pp.key = 'A' AND pp.key = 'B'`, `WHERE store_id LIKE '%city%'`, `SELECT *`).
- Name exact column, table, or key string involved.
- Generic phrasing like "avoid bad joins" or "check filters" rejected — must show concrete token.
- Pair offending pattern with corrective pattern in same rule.
