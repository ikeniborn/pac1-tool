## Grounding-Aware SQL Plan

Always ensure SQL plan projects `sku` when grounding required. `COUNT(*)` alone leaves answer phase without SKU values for `grounding_refs`.

- Count questions: pair `SELECT COUNT(*)` with companion `SELECT sku ... LIMIT 5` using identical WHERE.
- Existence/yes-no questions: project `sku` directly, not bare `SELECT 1`.
- Aggregate questions: include secondary query returning sample SKUs from underlying rows.
- Never emit final plan where no query returns `sku` column — answer phase cannot build `/proc/catalog/{sku}.json` refs.
- If LEARN diagnoses missing grounding, corrective rule must mandate `sku` projection in next plan cycle.
