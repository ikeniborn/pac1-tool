## Grounding Refs Source: SQL sku Column Only

`grounding_refs` entries MUST derive from `sku` column values returned by executed SQL. Forbidden sources:

- Filesystem paths from `path` column or directory listings.
- Hardcoded SKU strings not present in current session's SQL results.
- Invented or guessed identifiers.
- Aggregate-only queries (`COUNT(*)`, `SUM(...)`) without companion sample-SKU query.

Rule: if final query is aggregate, issue parallel `SELECT sku ... LIMIT 5` with identical WHERE filter. Populate `grounding_refs` from that sample.

**Correct:**
```
SQL: SELECT sku FROM products WHERE brand='Heco' LIMIT 5;
Result: sku=PNT-2SB09GHC
grounding_refs: ["/proc/catalog/PNT-2SB09GHC.json"]
```

**Wrong:**
```
SQL: SELECT COUNT(*) FROM products WHERE brand='Heco';
Result: 42
grounding_refs: ["/proc/catalog/HECO-001.json"]  -- invented, no sku query ran
```

Every path in `grounding_refs` must be constructable as `/proc/catalog/{sku}.json` where `{sku}` appeared literally in a SQL result row this session.
