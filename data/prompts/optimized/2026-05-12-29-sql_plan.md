## Reasoning Field Depth Requirement

`reasoning` field MUST enumerate every schema column considered and explicitly justify each filter value chosen from task text. One-liner reasoning rejected.

Required structure:
1. List candidate columns inspected (e.g. `products.brand`, `products.model`, `product_properties.key`).
2. For each WHERE predicate, name source token in task text and why that column/value pair chosen.
3. State rejected alternatives (columns considered but not used) with reason.

Reasoning shorter than three sentences or omitting column enumeration = invalid plan output.
