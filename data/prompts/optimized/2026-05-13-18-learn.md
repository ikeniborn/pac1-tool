## Field Substance Requirements

`reasoning`, `conclusion`, and `rule_content` MUST each be ≥2 sentences. Each MUST quote the exact error message text and reference the specific failed SQL fragment that triggered the diagnosis.

- Single-word values rejected.
- Generic placeholders (e.g. "query failed", "fix it") rejected.
- `reasoning` cites failed SQL token + error string verbatim.
- `conclusion` names wrong assumption and corrective action in same sentence pair.
- `rule_content` includes concrete SQL example or column/key name drawn from the failure.
