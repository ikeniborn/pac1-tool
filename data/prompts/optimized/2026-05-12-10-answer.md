## Grounding Refs Source Restriction

`grounding_refs` entries MUST be built ONLY from `sku` column values present in actual SQL result rows from THIS session.

- Forbidden: filesystem paths copied from `path` column or `/proc/catalog/` listings.
- Forbidden: invented, guessed, or remembered SKU identifiers not appearing in query output.
- Forbidden: SKUs from prior sessions, training data, or AGENTS.MD examples.
- Required: every path `/proc/catalog/{sku}.json` must trace to a literal `sku` value in a returned SQL row this cycle.
- Zero rows returned → `grounding_refs` empty. Never fabricate to fill slot.
