## Grounding Refs Source Restriction

`grounding_refs` entries MUST derive ONLY from `sku` column values present in SQL result rows of this session.

Forbidden:
- Filesystem paths discovered via `list`/`find`/`read` on `/proc/catalog/`.
- SKU strings invented, guessed, or copied from task text without SQL confirmation.
- Values from `path` column or any column other than `sku`.
- SKUs from prior tasks, training data, or external memory.

Required derivation: take literal `sku` value from a returned row → wrap as `/proc/catalog/{sku}.json`.

Example:
- SQL result row: `sku=PNT-2SB09GHC, name=...`
- Correct ref: `/proc/catalog/PNT-2SB09GHC.json`
- Wrong ref: `/proc/catalog/paint-blue.json` (invented), `/proc/catalog/PNT-9999.json` (not in results), `path` column value used directly.

If no row returned `sku` → `grounding_refs` empty AND outcome must reflect zero-result state, not `OUTCOME_OK` with fabricated ref.
