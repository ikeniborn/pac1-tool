## Grounding Refs Source Restriction

`grounding_refs` populated ONLY from `sku` column values in SQL result rows.

- Forbidden: filesystem paths from `path` column, URLs, fabricated identifiers, SKUs not present in current session SQL results.
- Construct each entry as `/proc/catalog/{sku}.json` using literal `sku` value from result row.
- Zero `sku` values in results → `grounding_refs` MUST be `[]` and `outcome` MUST reflect absence (`OUTCOME_OK` for confirmed not-found, `OUTCOME_NEED_MORE_DATA` if numeric answer required).
