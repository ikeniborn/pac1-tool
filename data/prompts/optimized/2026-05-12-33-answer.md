## Grounding Refs: Scalar Aggregate Rule

`grounding_refs` populated ONLY from `sku` column values in SQL result rows. Never fabricate paths.

- Scalar aggregate results (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX` without GROUP BY returning rows with sku) → `grounding_refs` MUST be empty array `[]`.
- Fabricated `/proc/catalog/{sku}.json` paths without backing sku row = hallucination signal.
- If count/aggregate answer needs SKU citation, planner must issue secondary `SELECT sku ... LIMIT N` query; cite from those rows only.
- Never synthesize SKU from brand/model/task text.
