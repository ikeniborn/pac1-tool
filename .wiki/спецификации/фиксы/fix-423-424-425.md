# FIX-423 / FIX-424 / FIX-425

**Дата:** 2026-05-01 | **Прогон-источник:** run 62.79% (27/43) | **Ожидаемый прирост:** +4–6 задач

## FIX-423 — Contract gate matching

**Задачи:** t01, t17, t26, t35

**Проблема:** `evaluator.py` проверяет, является ли `required_evidence` подстрокой `refs_str`. Но evaluator LLM генерирует дескриптивные строки (`"Read of /accounts/acct_001.json showing..."`), которые никогда не являются подстрокой голого пути `/accounts/`. Результат: каждый нетривиальный контракт → 2 reject → NONE_CLARIFICATION/DENIED вместо OK.

**Фикс** (`agent/evaluator.py`, строки 446–455): инвертировать проверку — evidence покрыто если хотя бы один grounding_ref является подстрокой строки evidence:
```python
# Было: e.lower() not in refs_str
# Стало: not any(ref.lower() in e.lower() for ref in refs)
```

## FIX-424 — Lookup evaluator bypass

**Задачи:** t30, t40

**Проблема:** `_should_bypass_evaluator_lookup()` возвращал `True` если было хотя бы одно exploration (list/read/search) — даже для OUTCOME_OK. Неверный ответ проходил без проверки.

**Фикс** (`agent/loop.py`): bypass только для OUTCOME_NONE_CLARIFICATION:
```python
def _should_bypass_evaluator_lookup(report) -> bool:
    """FIX-424: bypass evaluator for lookup only on NONE_CLARIFICATION."""
    return report.outcome == "OUTCOME_NONE_CLARIFICATION"
```

## FIX-425 — CRM date verification gate

**Задача:** t13

**Проблема:** CRM-правило требует TOTAL_DAYS = stated_days + 8. Evaluator (`mid` skepticism) не проверяет арифметику. Агент вычислил 14 дней вместо 14+8=22.

**Фикс** (`agent/loop.py`): pre-check до evaluator при `task_type == TASK_CRM` + `OUTCOME_OK`. Проверяет что в `completed_steps_laconic` есть явное упоминание VAULT_DATE и "+8" / "TOTAL_DAYS". Если нет → reject с hint о формуле.
