# Прогоны v1, v2, v3 — только целевые задачи (t40–t43, t13)

Три серии по 5 прогонов на модели `minimax-m2.7:cloud` (openrouter). Запускались только задачи `t40 t41 t42 t43 t13`. DSPy programs — `[missing]` во всех прогонах v1/v2; в v3 появился `prompt_builder_program.json`.

## Сводные результаты

| Версия | Стартовый граф | Средний балл | Лучший прогон |
|--------|---------------|--------------|---------------|
| v1 | 0 узлов | **16%** (4/25) | 20% (4 прогона по 1/5) |
| v2 | 78 узлов, 181 ребро | **12%** (3/25) | 20% |
| v3 | 41 узел, 50 рёбер | **16%** (4/25) | 40% (прогон 4: t43+t40) |

### v2 — Ключевые отличия от v1

- t43 прогоны 3/5: `DENIED_SECURITY` — возможно из-за накопленных antipatterns
- t13 прогоны 3/4: агент почти справился (дата off by 1), в прогоне 5 — `DENIED_SECURITY`
- Средний балл снизился: 16% → 12%

## Ключевые наблюдения

- **t41 (0/5 во всех трёх серий)**: формула gap+5 от VAULT_DATE возвращает неверный ESTIMATED_TODAY. Это баг в коде, не в промпте — накопленные знания не помогают.
- **t42 (0/5 везде)**: агент не может вычислить "N дней назад" от VAULT_DATE + найти статью по точной дате. Стабильно выдаёт CLARIFICATION или no answer.
- **t40 (нестабильно)**: успехи случайны — зависят от варианта vault (какой менеджер/компания). Граф накапливает правила про lookup, но агент всё равно пропускает файлы.
- **t13 (0/5)**: в v3 прогонах 3–4 агент почти справляется (находит файл, пишет дату), но ошибается на 8–20 дней.
- **Накопление не улучшает результаты** при сломанных postrun optimize — DSPy петля не замкнута.
