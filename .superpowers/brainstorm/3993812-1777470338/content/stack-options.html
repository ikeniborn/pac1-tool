A — Python-сервер + vis.js

FastAPI отдаёт граф через /api/graph, браузер рендерит через vis.js. Drag, zoom, hover-tooltips из коробки. Легко фильтровать на сервере.

Плюсы

  • Живые данные без перезагрузки
  • vis.js — лучший UX для графов
  • Просто добавить фильтры

Минусы

  • Нужен FastAPI/uvicorn
  • Две части: сервер + фронт

Standalone HTML
Python генерирует → открываешь в браузере

B — Генератор HTML-файла

Python-скрипт читает graph.json → записывает graph.html с встроенными данными и Cytoscape.js. Открываешь файл напрямую, без сервера.

Плюсы

  • Нет зависимостей-сервера
  • Файл можно шарить
  • Самый простой запуск

Минусы

  • Данные запечены в HTML
  • Нет live-reload

+ D3.js force-directed

C — stdlib сервер + D3.js

Минимальный сервер на http.server из stdlib (нет pip-зависимостей). D3.js force-directed граф с полным контролем над рендерингом.

Плюсы

  • Нет pip-зависимостей
  • D3 — максимум гибкости

Минусы

  • D3 сложнее в настройке
  • Больше кода для базовых вещей
