Узлы: цвет по типу · opacity по confidence · размер по uses · рёбра: requires

uv run python scripts/visualize_graph.py
→ FastAPI на :8765 → открываешь браузер
GET /api/graph → vis.js Network

A — FastAPI + vis.js ★ рекомендую

Один Python-файл запускает сервер. /api/graph возвращает JSON. vis.js рисует граф с drag, zoom, hover-tooltip. Фильтры по тегу/типу через параметры URL.

Плюсы

  • vis.js — лучший UX граф-рендеринг
  • live-reload данных (F5)
  • легко добавить фильтры на API

Минусы

  • fastapi + uvicorn в зависимостях
uv run python scripts/visualize_graph.py
→ http.server (stdlib) на :8765
HTML генерируется → Cytoscape.js

B — stdlib http.server + Cytoscape.js

Генерирует graph.html с данными, раздаёт через встроенный сервер. Cytoscape.js — специализирован для графов, отличный zoom/pan, много layout-алгоритмов.

Плюсы

  • нет pip-зависимостей
  • Cytoscape.js мощный для графов

Минусы

  • данные запечены при старте
  • нет live API
uv run python scripts/visualize_graph.py
→ записывает graph_viz.html
→ open graph_viz.html

C — Чистый HTML-файл (без сервера)

Скрипт генерирует самодостаточный scripts/graph_viz.html. Открываешь файл в браузере. Нет сервера, нет зависимостей — файл можно шарить.

Плюсы

  • самый простой запуск
  • portabl файл

Минусы

  • CORS-блокировка fetch в Chrome
  • нет live-обновлений

Рекомендую A — fastapi + uvicorn уже могут быть в зависимостях проекта (uv sync). vis.js лучший UX для интерактивного графа. Один файл scripts/visualize_graph.py делает всё.
